### 🧠 [QueryMind](https://github.com/2300030811/QueryMind)

> PPO-based RL agent that learns optimal PostgreSQL planner knob configurations, achieving up to 9.4x speedup over the default cost-based optimizer on TPC-H benchmarks.

[![Python](https://img.shields.io/badge/Python-3776AB?style=flat-square&logo=python&logoColor=white)](https://github.com/2300030811/QueryMind)
[![PostgreSQL](https://img.shields.io/badge/PostgreSQL-316192?style=flat-square&logo=postgresql&logoColor=white)](https://github.com/2300030811/QueryMind)
[![FastAPI](https://img.shields.io/badge/FastAPI-009688?style=flat-square&logo=fastapi&logoColor=white)](https://github.com/2300030811/QueryMind)
[![Docker](https://img.shields.io/badge/Docker-2496ED?style=flat-square&logo=docker&logoColor=white)](https://github.com/2300030811/QueryMind)
[![Stars](https://img.shields.io/github/stars/2300030811/QueryMind?style=flat-square&color=yellow)](https://github.com/2300030811/QueryMind/stargazers)
[![Last Commit](https://img.shields.io/github/last-commit/2300030811/QueryMind?style=flat-square)](https://github.com/2300030811/QueryMind)

**What it does:** Toggles 6 PostgreSQL planner knobs as a `Discrete(64)` action space. A PPO agent trained on TPC-H SF=1 learns which configurations produce fastest execution — hitting **9.41x speedup on Q17** and a 33% win rate over PostgreSQL default. Served via FastAPI with sub-50ms inference. Inspired by Bao, DQ, and Neo research.  
**Tech used:** `Stable-Baselines3 (PPO)` `Gymnasium` `PostgreSQL 17` `pg_hint_plan` `sqlglot` `FastAPI` `Docker` `W&B`

> Multi-agent market intelligence platform for Indian retail investors — 80+ swarm personality agents plus a 6-analyst specialist team producing plain-English signals for NSE and S&P 500 stocks.

[![Python](https://img.shields.io/badge/Python-3776AB?style=flat-square&logo=python&logoColor=white)](https://github.com/2300030811/AlphaHive)
[![Next.js](https://img.shields.io/badge/Next.js-000000?style=flat-square&logo=nextdotjs&logoColor=white)](https://github.com/2300030811/AlphaHive)
[![LangGraph](https://img.shields.io/badge/LangGraph-1C3C3C?style=flat-square&logo=langchain&logoColor=white)](https://github.com/2300030811/AlphaHive)
[![FastAPI](https://img.shields.io/badge/FastAPI-009688?style=flat-square&logo=fastapi&logoColor=white)](https://github.com/2300030811/AlphaHive)
[![Stars](https://img.shields.io/github/stars/2300030811/AlphaHive?style=flat-square&color=yellow)](https://github.com/2300030811/AlphaHive/stargazers)
[![Last Commit](https://img.shields.io/github/last-commit/2300030811/AlphaHive?style=flat-square)](https://github.com/2300030811/AlphaHive)

**What it does:** Layer 1 swarm simulates 80+ investor archetypes (Retail, Institutional, Algo, News Reactors) with two-round social influence logic, producing Bullish%, Panic Index, and Conviction Score. Layer 2 runs 6 specialist agents in parallel (Fundamental, Technical, Sentiment via FinBERT, News). A Bull/Bear debate synthesized by a Risk Manager produces a final explainable signal in plain English.  
**Tech used:** `LangGraph` `Ollama (LLaMA 3)` `FinBERT` `FastAPI` `PostgreSQL` `Redis` `Next.js` `Docker`
